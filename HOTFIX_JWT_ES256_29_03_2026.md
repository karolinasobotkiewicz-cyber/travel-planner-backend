# HOTFIX: JWT ES256 Support - 29.03.2026

## 🔴 Problem

Backend zwracał błąd podczas próby weryfikacji JWT tokenów:

```json
{
  "detail": "Invalid token: The specified alg value is not allowed"
}
```

**Przyczyna:** Backend był skonfigurowany tylko do weryfikacji tokenów HS256, ale Supabase w nowych projektach domyślnie używa ES256.

## ✅ Rozwiązanie

Zaktualizowano backend aby obsługiwał **oba algorytmy**:
- **ES256** (priorytet) - nowy standard Supabase, bezpieczniejszy (asymetryczna kryptografia)
- **HS256** (fallback) - stary standard, dla starszych projektów (symetryczna kryptografia)

Backend automatycznie wykrywa który klucz jest skonfigurowany i używa odpowiedniego algorytmu.

### 2. `app/infrastructure/auth/jwt_handler.py`

Przepisano funkcję `decode_jwt()` z logiką wykrywania algorytmu:

```python
def decode_jwt(token: str) -> Dict:
    # Określ który klucz/algorytm użyć
    if settings.supabase_jwt_public_key:
        # ES256 (rekomendowane): użyj klucza publicznego
        key = settings.supabase_jwt_public_key
        algorithms = ["ES256"]
    elif settings.supabase_jwt_secret:
        # HS256 (legacy): użyj współdzielonego sekretu
        key = settings.supabase_jwt_secret
        algorithms = ["HS256"]
    else:
        raise HTTPException(500, "JWT verification key not configured")
    
    # Zweryfikuj token
    payload = jwt.decode(token, key, algorithms=algorithms, ...)
    return payload
```

## 🔧 Konfiguracja dla Klientki

### Krok 1: Pobierz JWT Public Key z Supabase

1. Zaloguj się do **Supabase Dashboard**
2. Wybierz swój projekt
3. Przejdź do: **Settings** (⚙️) → **API**
4. Przewiń do sekcji **JWT Settings**
5. Znajdź pole **JWT Secret**
6. Jeśli widzisz tekst zaczynający się od `-----BEGIN PUBLIC KEY-----`, to twój projekt używa **ES256** ✅
7. **Skopiuj cały klucz** (włącznie z `-----BEGIN PUBLIC KEY-----` i `-----END PUBLIC KEY-----`)

**Ważne:** Klucz jest wieloliniowy. Po skopiowaniu będzie wyglądał tak:

```
-----BEGIN PUBLIC KEY-----
MFkwEwYHKoZIzj0CAQYIKoZIzj0DAQcDQgAEabcdefghijklmnopqrstuv
wxyz1234567890ABCDEFGHIJKLMNOPQRSTUVWXYZ
-----END PUBLIC KEY-----
```

### Krok 2: Zaktualizuj zmienne środowiskowe

#### Opcja A: Local Development (`.env`)

1. Otwórz plik `travel-planner-backend/.env`
2. Dodaj nową zmienną środowiskową:

```bash
SUPABASE_JWT_PUBLIC_KEY="-----BEGIN PUBLIC KEY-----\nMFkwEwYHKoZIzj0CAQYIKoZIzj0DAQcDQgAEabcdefg...\n-----END PUBLIC KEY-----"
```

**⚠️ Uwaga na formatowanie:**
- Cały klucz musi być w **jednym ciągu znaków**
- Zamień **każdy enter** (nową linię) na `\n`
- Otocz całość w **podwójne cudzysłowy** `"`

**Przykład transformacji:**

Z tego:
```
-----BEGIN PUBLIC KEY-----
MFkwEwYHKoZIzj0CAQYIKoZIzj0DAQcDQgAE
abcdefghijklmnopqrstuvwxyz
-----END PUBLIC KEY-----
```

Na to:
```bash
SUPABASE_JWT_PUBLIC_KEY="-----BEGIN PUBLIC KEY-----\nMFkwEwYHKoZIzj0CAQYIKoZIzj0DAQcDQgAE\nabcdefghijklmnopqrstuvwxyz\n-----END PUBLIC KEY-----"
```

## ❓ Pytania?

**Q: Dlaczego ES256 jest bezpieczniejszy niż HS256?**

A: ES256 używa **asymetrycznej kryptografii**:
- Supabase podpisuje tokeny **prywatnym kluczem** (tylko oni go mają)
- Nasz backend weryfikuje używając **publicznego klucza** (może być publicznie dostępny)
- Nawet jeśli ktoś ukradnie publiczny klucz, nie może podrobić tokenów

HS256 używa **symetrycznej kryptografii**:
- Ten sam sekret służy do podpisywania i weryfikacji
- Jeśli sekret wycieknie, atakujący może tworzyć fałszywe tokeny

**Q: Co jeśli nie wiem który algorytm używa mój Supabase?**

A: Sprawdź w Dashboard → Settings → API → JWT Settings:
- Jeśli widzisz `-----BEGIN PUBLIC KEY-----` → masz **ES256** ✅
- Jeśli widzisz losowy ciąg znaków (np. `pvaAG1JoRNPi...`) → masz **HS256**

**Q: Backend nadal zwraca 401 po konfiguracji**

A: Sprawdź:
1. Czy klucz jest poprawnie sformatowany z `\n`
2. Czy nie ma spacji na początku/końcu klucza
3. Czy backend się zrestartował po zmianie zmiennej
4. Czy w logach nie ma błędów parsowania klucza

**Q: Mogę używać obu kluczy jednocześnie?**

A: Backend będzie używać **tylko jednego**:
- Jeśli `SUPABASE_JWT_PUBLIC_KEY` istnieje → używa ES256
- Inaczej, jeśli `SUPABASE_JWT_SECRET` istnieje → używa HS256
- Jeśli żaden nie istnieje → zwraca 500 error

Zalecamy używać **tylko ES256** dla nowych projektów.
