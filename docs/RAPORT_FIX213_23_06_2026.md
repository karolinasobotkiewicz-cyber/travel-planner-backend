## Obszary naprawione w silniku

### 1. Coverage API (fałszywe „pokryte”)
- **Planty / Maczuga / Muzeum Motyla** nie liczą się już jako `nature_landscape`
- **Browar / Wedel / Pijalnia** nie liczą się jako `active_sport` ani `water_attractions`
- **Muzeum Lotnictwa, Łazienki (jako woda), Bulwary (jako woda)** — poprawione
- **Katedra** → `history_mystery`, nie `museum_heritage`
- **Próg adequacy**: `nature_landscape` / `relaxation` wymagają **≥2 mocnych POI** (nie wystarczy 1 Planty)

### 2. Scoring / wybór POI
- Boost: **Kopiec, Bulwary, Zakrzówek, Łazienki, Ogród botaniczny** przy nature/relax
- Kara: **Planty, Maczuga** gdy user chce naturę/relaks
- Boost: **Hydropolis, Kolejkowo** przy profilu rodzinnym
- **Adventure + active_sport**: silniejsza kara muzeów (70%), wyłączony cultural_bonus, kara browarów/Wedel
- **must_see** słabsze gdy POI nie pasuje do preferencji (×0.5 w city tourism)

### 3. Transit / geografia
- City tourism: **walk threshold 3.5 km**, prędkość **4.5 km/h** (realistyczne przejścia w centrum)
- **POI >28 km** od centrum miasta — tylko późne dni / Zone C (Wojsławice, Maczuga poza centrum)
- **Zone A link_km = 5** — ciaśniejsze grupowanie centrum (Kraków Stare Miasto)
- **cluster_krakow_core** — mniej rozproszenia Wawel/Rynek/Podziemia na wiele dni

### 4. Długie plany (5–7 dni)
- **FIX #160**: ostatnie 2 dni — retry z pełnej puli POI gdy <3 atrakcje
- **Guliwer** — blokada dla couples/friends/solo (tylko family_kids)

### 5. Tag mapper
- `brewery`, `craft_beer`, `chocolate_tasting` → **local_food** (nie active_sport)
- `outdoor` / `entertainment` — bez fałszywego `active_sport`
