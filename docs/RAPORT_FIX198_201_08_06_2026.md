## 4. Szczegóły techniczne FIX #201

### 4.1 Cluster hub day pools

```python
# city_copy.py
build_cluster_hub_day_pools(all_pois, num_days, cluster_cities)
# → pools[d] = POI tylko z hub przypisanego do dnia d
```

Log w konsoli: `[FIX #201] Cluster day N → hub 'Gdynia' (42 POI)`

### 4.2 Quick-stop denylista

POI z nazwami typu Bamberki, Długi Targ, deptaki:
- Nie dostają `must_see` / `core_attraction` badge
- Nie dostają iconic scoring boost (`must_see >= 8`)

### 4.3 Explainability

Przed: luźne dopasowanie tagów → „Relaxing activity” przy `relaxation: false`  
Po: `poi_covers_preference_report()` — identyczna logika jak `preference_coverage` w API

### 4.4 Dinner pass

Kolejność w pipeline dnia:
1. afternoon top-up (#195)
2. heal timeline (#196)
3. **enforce dinner (#201)** ← NOWE
4. day_end enforcer
5. append `DayEndItem`
