czasem generuje duże bloki `free_time` (np. 4+ godzin). To nie jest błąd
nowych FIXów — to pre-existing zachowanie silnika.

### 2. Linia 4116 engine.py — niezałatany gap FIX #65
```python
# STARY KOD (nie naprawiony):
lunch_restaurants = [r for r in restaurants_available if r.get("meal_type") == "lunch"]
# POWINNO BYĆ:
lunch_restaurants = [r for r in restaurants_available if "lunch" in (r.get("meal_type") or "").lower()]
```

Ten kod jest w bloku `# Get lunch suggestions (same logic as main lunch block)`.
Nie powoduje błędów w obecnych testach, ale może pominąć restauracje
`meal_type="lunch,dinner"` w edge-case'ach.
