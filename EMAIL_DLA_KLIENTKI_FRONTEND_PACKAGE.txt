KLUCZOWE ENDPOINTY ETAPU 2:

POST /payment/create-checkout-session (wymaga auth)
↳ Tworzy sesję Stripe Checkout
↳ Zwraca checkout_url (redirect) i session_id (do pollowania)

GET /payment/session/{id}/status (wymaga auth)
↳ Sprawdza status płatności
↳ payment_status: pending, paid, unpaid, expired
↳ Pollować co 3-5s max 60s po redirect ze Stripe

POST /plan/claim-guest-plans (wymaga auth)
↳ Transferuje plany guest → authenticated user
↳ Request: { "guest_id": "uuid-from-localstorage" }
↳ Response: { "success": true, "transferred_plans": 3 }

GET /plan/my-plans (wymaga auth)
↳ Lista wszystkich planów użytkownika
↳ Dla "My Plans" dashboard page

---

POLLING STRATEGY (gotowy kod w sample responses):
• Interwał: 3-5 sekund
• Max czas: 60 sekund (12-20 prób)
• Po timeout: Pokaż "Sprawdź później" + link do manualnej weryfikacji
• Kiedy pollować: Po redirect ze Stripe Checkout

Status transitions:
pending → paid (sukces! pokaż pełny plan)
pending → unpaid (płatność nieudana, pokaż error)
pending → expired (sesja wygasła po 24h)
